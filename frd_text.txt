Introduction ProjectSphere is an all-in-one project management platform built to connect daily team tasks with high-level business goals. It moves away from scattered tools by bringing project tracking, team attendance, and performance evaluations into a single, professional workspace 1.1 Purpose This document describes, in simple business language, what the ProjectSphere web application does. It explains the features available to each type of user and shows how each screen looks in the live system. The document is meant to be read by business stakeholders, project managers, developers and reviewers without needing technical background. 1.2 Intended Audience Admins: Who need to manage global data, upload bulk records, and oversee every project from a bird's-eye view. Project Managers: Who need to evaluate team performance and track health metrics across multiple projects. Developers & Trainees: Who need a functional, Jira-style dashboard to manage their daily stories, tasks, and defects. 2. Functional Requirements ProjectSphere is organized around three user roles: Administrator, Project Manager and Developer. Every user signs in with their email and password. Based on their role, the system takes them to the right home page and shows only the screens that are relevant to them. The sections below describe each area of the application in plain language. 2.1 Login Login is the entry point of the application. Users enter their email and password to access the system. If a user forgets their password, they can reset it from the same screen. 2.1.1 Sign In A signed-in user is taken to the dashboard that matches their role. Wrong credentials show a clear error message. Each user only sees the menus and actions that match their role. Key things you can do: Enter email and password to sign in. See an error message if the email or password is wrong. Get redirected to the right dashboard based on the role (Admin, PM or Developer). Figure 2.1.1: Sign-in screen 2.1.2 Forgot Password A self-service way for users to reset their own password without involving the administrator. Key things you can do: Open the Forgot Password option from the login screen. Set a new password for the registered email. 875281 310610 0 0 Sign in immediately with the new password. Figure 2.1. 2 : Reset password screen 2.2 Administrator Area T he Administrator owns the workspace. The administrator sets up the projects, the project managers, the team members and the teams. Once the basics are in place, project managers and developers can start working without further setup. 2.2.1 Admin Dashboard The home page for the administrator. It shows the overall picture of the workspace at a glance and provides shortcuts to every management area. Key things you can do: See the total number of projects, project managers, team members and teams. View quick health indicators across the organization. Open any management area (projects, managers, members, teams) from one place. Figure 2.2.1: Administrator dashboard 2.2.2 Manage Projects A central list of every project in the organization. The administrator creates new projects, edits existing ones, or removes projects that are no longer needed. Key things you can do: View all projects with their key details (name, owner, status). Create a new project. Edit an existing project (rename, change owner, update details). Delete projects that are no longer required. Figure 2.2.2: Projects list (Admin view) 2.2.3 Manage Project Managers A directory of all project managers. The administrator decides who can act as a PM and on which projects. Key things you can do: See the full list of project managers and their contact details. Add a new project manager to the system. Edit a project manager’s profile or activate/deactivate them. Assign or change the projects a PM is responsible for. Figure 2.2.3: Project Managers list 2.2. 4 Manage Teams Teams group people together to deliver a project. The administrator creates teams and decides who is in them. Key things you can do: View all teams in the organization. Create a new team and add members to it. Edit a team (rename, add/remove members, change the lead). Disband a team that is no longer needed. Figure 2.2.5: Teams list (Admin view) 2.2.5 Add Team Member From the Admin Dashboard, the administrator can register a new team member. Two options are provided on a single dialog: a bulk upload of an Excel (.xlsx) file for onboarding many people at once, or a manual single-record form to register one person. Key things you can do: Bulk upload a .xlsx file containing many team members and register them in one click. Manually register one team member by filling in Employee ID, Phone Number, First Name, Last Name, Email and Password. Close the dialog without saving when needed.
